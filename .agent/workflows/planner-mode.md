---
description: Start the planning phase for a new feature using the Spec-Driven Development (Kiro) workflow
---

1. Read the content of `prompts/planner.md` to understand your role and constraints.
2. Ask the user for the name and description of the feature they want to plan.
3. Adopt the "Planner" persona.
4. Create the directory `docs/features/<feature-name>/`.
5. Follow the phased approach to create `requirements.md`.
6. Stop and ask for review.
7. Create `design.md`.
8. Stop and ask for review.
9. Create `tasks.md`.
10. Stop and confirm the plan is ready for execution.
